The constructor function's floating pragma was set to a specific version, to avoid future Solidity non-backward compatible changes which may affect the contract functionality. 

In addition, the function massHarvest was fixed to eliminate the DoS with Block Gas Limit. In Ethereum, gas is used to limit the amount of work that can be performed in a block and avoid DoS attacks. However, if a large array is iterated over, the total gas required can exceed the block gas limit, causing the transaction to fail. In this case, a large array of unharvested epochs might cause this. To solve this problem, the logic was changed to a pagination approach where you can limit the iterations to a maximum of 100 epochs to avoid exceeding the block gas limit.
