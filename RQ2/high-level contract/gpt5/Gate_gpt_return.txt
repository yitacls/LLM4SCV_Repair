An attacker (or an honest user using a contract account) can therefore block the admin from settling multiple orders by creating an order whose owner is a contract that causes transfer to fail, causing a denial-of-service for processing the remaining orders.\n\nImpact:\n- Admin cannot complete multiFill if any recipient's transfer reverts, preventing settlement of other orders and locking funds until the problematic order is removed.\n- This can be used to grief the system or stall withdrawals/fulfillments.\n\nExploit steps:\n1.
