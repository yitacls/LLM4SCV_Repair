    "description": "The modifier uses tx.origin for ownership checks (require(tx.origin == owner,...)). Relying on tx.origin for authorization is insecure because tx.origin refers to the original externally-owned account that started the transaction and is preserved across calls through intermediate contracts. An attacker can create a malicious contract that lures the owner (EOA) to call it; that malicious contract can then call BridgeOperatable functions. Since tx.origin will still equal the owner's EOA, the require check passes and the attacker-controlled contract can execute owner-only functionality, effectively bypassing access control. The code comment suggests this may be an accidental replacement of the safer check require(msg.sender == owner, ...). Remediation: do not use tx.origin for access control. Use msg.sender for direct caller checks (e.g., require(msg.sender == owner,...)) or adopt a well-audited ownership/access-control implementation such as OpenZeppelin's Ownable. If you intended to restrict only EOAs, explicitly reason about that pattern and its trade-offs; avoid tx.origin-based checks. Severity: High/Critical."
