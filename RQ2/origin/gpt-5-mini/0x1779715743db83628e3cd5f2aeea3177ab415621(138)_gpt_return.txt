  "description": "The function uses tx.origin for authorization (require(tx.origin == owner())), which is insecure. An attacker can deploy a malicious contract and induce the owner (an EOA) to call that malicious contract; when the malicious contract calls CardSale.addMeByRC(), tx.origin still equals the owner's EOA so the require passes, but msg.sender will be the malicious contract address. Because the code sets rc[msg.sender] = true, the attacker (or their contract) gains RC privileges. This is a classic tx.origin phishing/access-control vulnerability. Fixes: do not use tx.origin for access control — use msg.sender == owner() or the Ownable onlyOwner modifier for owner-only checks. If the intent is for the owner to add other addresses, require(msg.sender == owner()) and accept an address parameter to grant rc[address] = true explicitly."
